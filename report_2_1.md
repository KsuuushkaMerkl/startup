# 1. Идея, проблема и сценарий

## Идея  
Маркетплейс микростажировок: компании размещают короткие оплачиваемые задачи с депозитом, кандидаты получают доступ после теста, оплата — только за выбранное решение.

## Проблема  
- Соискателям сложно получить коммерческий опыт без риска ошибки  
- Компаниям дорого и долго запускать стажировки без гарантии результата  

## Сценарий  

**Пользователь/роль:**  
- Джун/студент  
- Тимлид/PM  

**Ситуация:**  
- Джун ищет первый опыт  
- Компания хочет быстро решить небольшую задачу  

**Проблема:**  
- Джун не проходит фильтры без опыта  
- Компания не может быстро и объективно выбрать исполнителя  

**Последствия:**  
- Соискатель: потеря времени, отсутствие портфолио  
- Компания: затраты времени, риск слабого результата

# 2. Сегментация рынка

## Оси сегментации
- Тип клиента: B2B / B2C  
- Размер компании: SMB / Enterprise  
- Тип задач: типовые / сложные  

## Сегменты

### S1 — SMB IT (10–200 сотрудников)
- Кто принимает решение: тимлид/PM  
- Проблема: частые небольшие задачи  
- Текущее решение: фриланс  
- Ограничения: бюджет, скорость  
- Каналы: LinkedIn, Telegram, VC.ru  

### S2 — Enterprise
- Кто принимает решение: HR + руководители  
- Проблема: масштабный найм  
- Текущее решение: стажировки  
- Ограничения: бюрократия  
- Каналы: HR-сообщества  

### S3 — Не IT компании  
- Кто принимает решение: IT отдел  
- Проблема: нет ресурсов на стажировки  
- Текущее решение: аутсорс  
- Ограничения: бюджет, контроль качества  
- Каналы: бизнес-сообщества  

### S4 — Агентства  
- Кто принимает решение: руководители проектов  
- Проблема: пики загрузки  
- Текущее решение: фриланс  
- Ограничения: контроль качества  
- Каналы: LinkedIn, Telegram, VC.ru

# 3. Выбор приоритетного сегмента

## Приоритет: S1 (SMB IT)

- Высокая частота задач  
- Быстрое принятие решений

# 7. Ценностное предложение

**Формулировка:**

Для SMB IT-команд, которым нужно быстро решить небольшую задачу, мы даём возможность получить несколько решений и выбрать лучшее, снижая риски и время по сравнению с фрилансом.

**Почему это работает:**
- Входной тест  
- Депозит  
- Конкурс решений

# 9. Гипотезы

1. Малый и средний бизнес имеют ≥2 задач в месяц  
2. Компании готовы платить за лучший результат  
3. Джуны готовы участвовать  
4. Тест повышает качество  
5. Конкурс лучше фриланса  
6. Telegram — основной канал  
7. Конверсия ≥30%  
8. Депозит приемлем  
9. Основной барьер — доверие  
10. Средний чек: 2000-10000  

---

# 10. ТОП-3 риски

## 1. Готовность платить
- Метод: интервью  
- Критерий: ≥3/10  

## 2. Готовность джунов
- Метод: MVP  
- Критерий: ≥30% участие  

## 3. Конкурс и фриланс
- Метод: эксперимент с 1–3 компаниями (одна задача → 2 формата: фриланс и конкурс решений)  
- Критерий: лучший результат
